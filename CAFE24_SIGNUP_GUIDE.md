# 카페24 가입 및 초기 설정 가이드

**목표:** 카페24에서 Node.js 호스팅 + PostgreSQL 데이터베이스 환경 구축

---

## 🔑 Step 1: 카페24 회원가입

### 1-1. 카페24 방문
```
URL: https://www.cafe24.com/
```

### 1-2. 회원가입 절차
```
1. 우측 상단 "회원가입" 클릭
2. 개인 또는 사업자 선택 (개인 선택 권장)
3. 이메일 입력
4. 비밀번호 설정
   - 8자 이상, 대소문자 + 숫자 + 특수문자 포함
5. 개인정보 입력
   - 이름
   - 휴대폰 번호
6. 약관 동의
   - 필수 항목 모두 체크
7. "완료" 버튼 클릭
```

### 1-3. 이메일 인증
```
1. 등록한 이메일 확인
2. 카페24 인증 메일 클릭
3. 이메일 인증 완료

⏱️ 시간: 5-10분
```

## 🌐 Step 3: 도메인 설정

### 3-1. 도메인 선택 옵션

**옵션 A: 카페24에서 신규 도메인 구매 (추천)**
```
장점:
- 카페24에서 자동 관리
- DNS 자동 연동
- SSL 인증서 자동 적용

단점:
- 도메인 비용 추가

가격: 연 10,000~30,000원 (도메인에 따라)

예시 도메인:
- aedcheck.kr: 연 10,000원
- aedsystem.com: 연 15,000원
- 기타 원하는 도메인
```

**옵션 B: 기존 도메인 이전**
```
장점:
- 기존 도메인 재사용
- 이미지 유지

단점:
- 이전 프로세스 필요
- DNS 수동 설정

준비물:
- 기존 도메인 등록처 (Namecheap, GoDaddy 등)
- 도메인 관리자 정보
- 인증 코드 (도메인 이전 시 필요)
```

**옵션 C: 도메인 없이 IP로 접근**
```
장점:
- 비용 절감

단점:
- 주소 기억하기 어려움
- 이메일 사용 불가
- SEO 불리

예: http://123.45.67.89:3000

⚠️ 비추천
```

### 3-2. 도메인 구매 절차 (옵션 A 선택 시)

```
1. 카페24 관리자 패널 로그인
2. 좌측 메뉴 > "도메인"
3. "도메인 검색" 클릭
4. 원하는 도메인명 입력
   예: aedcheck.kr
5. 가용성 확인
   ✅ 신청 가능 → "장바구니 추가" 클릭
   ❌ 이미 등록됨 → 다른 도메인 시도
6. 장바구니에서 결제
   - 결제 수단 선택 (신용카드, 계좌이체)
   - 결제 완료

⏱️ 시간: 30분 ~ 1시간
💰 비용: 10,000~30,000원/년
```

### 3-3. 도메인 이전 절차 (옵션 B 선택 시)

```
1. 기존 도메인 등록처에서 인증 코드 받기
   - Namecheap: Account > Manage > Domain > AuthCode
   - GoDaddy: 같은 방식
   - 기타: 도메인 등록처 고객센터 문의

2. 카페24에서 도메인 이전 신청
   - 관리자 패널 > "도메인" > "도메인 이전"
   - 도메인명 입력
   - 인증 코드 입력
   - 결제 (보통 연장비용만 청구)

3. 이전 승인 대기
   - 보통 2~3일 소요
   - 기존 등록처에서 이전 승인 요청

4. DNS 설정
   - 카페24 관리자 패널에서 DNS 변경
   - 반영 시간: 24~48시간

⏱️ 시간: 2~3일
💰 비용: 5,000~15,000원 (연장비용)
```

**추천: 옵션 A (신규 도메인 구매)**
- 가장 간단
- 자동으로 모든 것 연결됨

## 💾 Step 4: PostgreSQL 데이터베이스 신청

### 4-1. 호스팅 Pro에 포함된 PostgreSQL

**호스팅 Pro 선택 시 이미 포함:**
```
✅ PostgreSQL 데이터베이스
  - 용량: 1GB~10GB (상품에 따라)
  - 사용자 수: 무제한
  - 백업: 자동 백업

추가 신청 필요 없음!
```

### 4-2. 데이터베이스 접근 정보 확인

```
호스팅 신청 후 3~4시간 후:

1. 카페24 관리자 패널 로그인
2. 좌측 메뉴 > "데이터베이스" > "PostgreSQL"
3. 다음 정보 확인 및 메모:

필요한 정보:
□ Host (호스트명): 예) pgsql.cafe24.com
□ Port (포트): 보통 5432
□ Database (DB명): 예) userid_dbname
□ Username (사용자명): 예) userid
□ Password (비밀번호): 예) ******* (초기 설정 시 생성)

이 정보들은 아주 중요합니다!
📌 메모장에 저장해두세요
```

### 4-3. PostgreSQL 초기 비밀번호 설정 (첫 접속 시)

```
1. 관리자 패널 > "데이터베이스" > "PostgreSQL"
2. "비밀번호 변경" 클릭
3. 새로운 비밀번호 입력
   - 8자 이상
   - 대소문자 + 숫자 + 특수문자 포함
4. "변경" 클릭

강력한 비밀번호 예:
AedSystem@2025Db!
```

### 4-4. 로컬 환경에서 연결 테스트

```bash
# 1. PostgreSQL 클라이언트 설치 (맥)
brew install postgresql

# 2. 연결 테스트
psql -h pgsql.cafe24.com \
     -U userid \
     -d userid_dbname \
     -p 5432

# 프롬프트가 나타나면 성공!
# 비밀번호 입력: (위에서 설정한 비밀번호)

# 3. 연결 확인 명령어
userid_dbname=# SELECT version();

# 출력 예:
# PostgreSQL 13.0 on x86_64-...
```

**연결 실패 시 확인사항:**
```
□ Host/Port 정확한가?
□ Username/Password 정확한가?
□ 방화벽이 5432 포트를 막고 있지 않은가?
□ 카페24 기술 지원팀에 문의
```

## 🛠️ Step 5: 호스팅 관리자 패널 설정

### 5-1. 관리자 패널 접근

```
URL: https://www.cafe24.com/admin
또는
로그인 후 "관리자 패널" 클릭
```

### 5-2. 필수 설정 항목

#### 5-2-1. FTP/SFTP 사용자 추가

```
좌측 메뉴 > "파일 관리" > "FTP 계정"

1. "FTP 계정 추가" 클릭
2. 새로운 FTP 사용자명 입력
   예: aedcheck_user
3. 비밀번호 설정
4. 홈 디렉토리: /
5. "저장" 클릭

저장될 정보:
□ FTP Host: 예) ftp.cafe24.com
□ Username: aedcheck_user
□ Password: (위에서 설정)
□ Port: 21 (또는 990 for SFTP)
```

#### 5-2-2. 웹 문서 루트 확인

```
좌측 메뉴 > "호스팅 정보" 또는 "설정"

확인사항:
□ 웹 문서 루트: /home/userid/public_html
□ 웹 접근 URL: http://yourdomain.kr
□ Node.js 포트 (기본): 3000 또는 8080

중요: Node.js 앱은 일반적으로 포트 3000에서 실행
```

#### 5-2-3. SSL 인증서 설정

```
좌측 메뉴 > "SSL/TLS" > "Let's Encrypt"

1. "인증서 발급" 클릭
2. 도메인명 확인
3. "발급" 클릭
4. 몇 분 대기 (자동으로 적용됨)

확인:
https://yourdomain.kr 접속해서 🔒 표시 있는지 확인
```

#### 5-2-4. 백업 설정

```
좌측 메뉴 > "백업" > "자동 백업 설정"

설정:
□ 자동 백업: 활성화 ✅
□ 백업 주기: 일일 (매일 자동 백업)
□ 보관 기간: 30일 이상
□ 데이터베이스: PostgreSQL 포함 ✅

이렇게 설정하면 매일 자동 백업됨
```
